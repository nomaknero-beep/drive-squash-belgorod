# Сайт сквош-клуба «Драйв» (Белгород) — дизайн-документ

Дата: 2026-06-24

## Цель

Сделать современный сайт для реального сквош-клуба «Драйв» (Белгород), который
будет лучше и функциональнее двух референсов (citysquash.ru, squashclub.moscow):
красивее, быстрее, с интерактивным бронированием и геймификацией.

## Технический стек

- **Фронтенд:** статика — HTML + CSS + ванильный JavaScript. Без тяжёлых
  фреймворков ради скорости загрузки и простоты размещения.
- **Бэкенд:** PHP (приём заявок брони, отправка в Telegram, выдача данных рейтинга).
- **БД:** MySQL (рейтинг игроков, турниры). Заявки на бронь НЕ хранятся в БД —
  только пересылаются админу в Telegram.
- **Хостинг:** RuCenter. Стек PHP+MySQL работает и на обычном (shared), и на VPS —
  выбор хостинга не блокирует разработку. Конкретный тип уточним позже вместе.

## Структура сайта

Основная страница — одностраничник (лендинг) с якорной навигацией, плюс
динамические данные рейтинга/турниров.

1. **Hero** — крупный экран, фото/видео кортов, заголовок, две кнопки:
   «Записаться» и «Бесплатная пробная по субботам».
2. **Преимущества** — карточки: 2 корта, прокат инвентаря, тренеры, удобное место.
3. **Бронирование** ⭐ — интерактивный календарь:
   шаги дата → время → корт → тренер (опц.) → имя + телефон → отправка.
   Заявка уходит админу в Telegram. Подтверждение клиенту: «заявка принята,
   с вами свяжутся». Реальная занятость кортов НЕ проверяется — подтверждает
   админ вручную.
4. **Цены и абонементы** — карточки тарифов.
5. **Тренеры и расписание групп** — фото, имена, время групп.
6. **Рейтинг и турниры** ⭐ — таблица рейтинга игроков (из MySQL) + анонсы и
   результаты турниров. Геймификация: места, очки, движение в таблице.
7. **Галерея** — фото клуба.
8. **Контакты** — карта, телефон, ВК, Telegram, часы работы.

## Ключевые фишки (отличие от конкурентов)

- Интерактивный пошаговый календарь бронирования с мгновенной заявкой в Telegram.
- Тёмная/светлая тема с плавными анимациями появления секций, премиальный вид.
- Рейтинг игроков и турнирная таблица (геймификация) — у конкурентов этого нет
  в таком виде.

## Потоки данных

### Бронирование
1. Клиент проходит шаги календаря на фронте (JS).
2. JS отправляет POST на PHP-эндпоинт (`/api/booking.php`) с данными заявки.
3. PHP валидирует и вызывает Telegram Bot API (sendMessage) с токеном бота,
   шлёт сообщение в чат админа (chat_id).
4. PHP возвращает фронту статус. Фронт показывает «заявка принята».

### Рейтинг/турниры
1. Данные хранятся в MySQL (таблицы players, tournaments).
2. PHP-эндпоинт (`/api/rating.php`) отдаёт JSON.
3. JS рисует таблицу. Наполнение/правка данных — вручную владельцем
   (простая защищённая страница админки или прямой импорт — уточнить).

## Telegram-бот

Создаём с нуля: инструкция по @BotFather (получить токен), узнать chat_id админа.
Токен и chat_id кладём в серверный конфиг PHP (не в публичный фронтенд-код).

## Компоненты (единицы с одной ответственностью)

- `index.html` — разметка лендинга.
- `css/styles.css` — стили, темизация (CSS-переменные для dark/light).
- `js/theme.js` — переключатель темы.
- `js/animations.js` — анимации появления секций (IntersectionObserver).
- `js/booking.js` — логика пошагового календаря и отправка заявки.
- `js/rating.js` — загрузка и отрисовка рейтинга/турниров.
- `api/booking.php` — приём заявки и отправка в Telegram.
- `api/rating.php` — выдача данных рейтинга из MySQL.
- `api/config.php` — секреты (токен бота, доступ к БД), вне публичного доступа.
- `db/schema.sql` — структура таблиц MySQL.

## Обработка ошибок

- Бронь: при недоступности Telegram API — показать клиенту запасной контакт
  (телефон) и не терять заявку (лог на сервере).
- Валидация телефона/имени на фронте и на сервере.
- Рейтинг: при сбое БД — показать заглушку, не ломать страницу.

## Тестирование

- Ручная проверка формы брони → реальное сообщение в Telegram.
- Проверка адаптива (мобильный/десктоп).
- Проверка переключения темы.
- Проверка отрисовки рейтинга с тестовыми данными.

## Вне scope (YAGNI)

- Онлайн-оплата (эквайринг).
- Личные кабинеты клиентов.
- Реал-тайм занятость кортов / автоматическое подтверждение.
- Поиск партнёра по игре (отложено).
- Бадминтон/настольный теннис (клуб только сквош).

## Открытые вопросы

- Тип хостинга RuCenter (shared/VPS) — проверим вместе.
- Реальные цены, тренеры, расписание, фото, логотип — предоставит владелец.
- Способ наполнения рейтинга (мини-админка или ручной импорт).
